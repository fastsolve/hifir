We choose to use STL list because adding new node is constant time without bothering copying. It's worth noting that querying the size of list has become constant since C++11. Also, with the constructors in Prec, the following way can be used to construct list:

using precs_t = Precs<double, int>;
using prec_t = precs_t::value_type;
precs_t precs;
precs.emplace_back(m, n, /* rvalue references */);

Regarding preconditioner, C++ generic programming is used, i.e. the member interfaces of Prec are implicitly assumed. Also, for most cases, the work array should be size of n, which is the system size of first level. However, if we have too many levels, the size of work may not be system size. Since we use a generic array interface for the buffer type, it's not easy to check out-of-bound accessing, as a result, it's critical to ensure the size of work buffer. Make sure work size is no smaller than the size given by compute_prec_work_space
