The C Library libhifir

The C library libhifir is designed to have prefix "lhf" for all interfaces. For constant variables, prefix "LHF_" is used; for types, prefix "Lhf" is used; for functions, lower case prefix "lhf" is used. For some types and functions, data types are put after prefix, and the data type conventions are the same as those used in BLAS/LAPACK. In summary, for most types and functions, we have the convection of "lhf[dszc]DoSomething" where d, s, z, and c stand for double real, single real, double complex, and single complex values.

Common Interfaces

  • LHF_NUMBER_PARAMS Length of control parameter array, i.e., construct a parameter array with double params[LHF_NUMBER_PARAMS].
  • LhfInt This is the integer type used in libhifir. It is 32-bit integer if linking to libhifir.so; it is 64-bit integer if linking to the 64-bit library libhifir_i64.so.
  • LhfIndPtr This is the integer type for ind_start arrays in sparse matrices as well as in HIF. Note that LhfInt is for index arrays in sparse matrices and HIF. By default, LhfIndPtr is ptrdiff_t
  • LhfStatus
  • LhfOperationType
  • lhfGetVersions
  • lhfEnableWarning
  • lhfDisableWarning
  • lhfGetErrorMsg If the return status is LHF_HIFIR_ERROR, then call this function to get the detailed message about the error.
  • lhfSetDefaultParams
  • lhfSetDroptol Set drop tolerance for both L and U factors.
  • lhfSetAlpha Set fill-in factors for both L and U factors.
  • lhfSetKappa Set conditioning thresholds for both D and L/U.

Standard APIs

Note that we only list APIs for double-precision real arithmetic. Everything in this section is applicable to other data types by replacing d (double real) by s (single real), z (double complex), and c (single complex).

  • LhfdMatrixHdl Matrix handle
  • LhfdHifHdl HIF handle
  • lhfdCreateMatrix Construct LhfdMatrixHdl
  • lhfdDestroyMatrix
  • lhfdGetMatrixSize and lhfdGetMatrixNnz
  • lhfdWrapMatrix Wrap external matrix data
  • lhfdCreate Construct a HIF instance of type LhfdHifHdl
  • lhfdDestroy
  • lhfdSetup Set up a HIF handle
  • lhfdUpdate Update the coefficient matrix
  • lhfdRefactorize Factorize a new preconditioner given a sparsifer
  • lhfdApply Apply a preconditioner with different operations, see LhfOperationType.
  • lhfdSolve Triangular solve, which is equivalent to calling lhfdApply with operation LHF_S where "S" stands for "Solve."
  • lhfdGetStats, lhfdGetNnz, lhfdGetLevels, lhfdGetSchurSize, and lhfdGetSchurRank

Mixed APIs

It is also possible to enable mixed-precision computation in libhifir.

  • lhfsdUpdate This function updates a single HIF handle with double precision coefficient matrix LhfdMatrixHdl.
  • lhfsdApply This function applies a single HIF to double-precision arrays for both input and output.
  • lhfsdSolve

Note that replace "sd" by "cz" for mixed-precision computations for complex arithemtic.
