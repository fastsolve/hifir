Detailed Description

template<class ValueType, class IndexType, template< class > class UserDenseFactor = DefaultDenseSolver>
class hif::HIF< ValueType, IndexType, UserDenseFactor >

Template Parameters
ValueTypenumerical value type, e.g. double
IndexTypeindex type, e.g. int
UserDenseFactorPotential user customized dense factor

This is top user interface (C++); it is designed as a preconditioner that can be easily plugin other codes. There are two core member functions, 1) factorize a multilevel ILU preconditioner and 2) solve the preconditioner system. For computing preconditioner, the input can be either CCS or CRS, for solving, the input must be Array. However, be aware that both CCS/CRS and Array can be used as external data wrappers, thus one should not worry about input data duplications.

The following is a simple workflow for Builder.

#include <hifir.hpp>
using namespace hif;
using builder_t = HIF<double, int>;
using crs_t = builder_t::crs_type;
int main() {
const auto A = wrap_const_crs(...);
builder_t builder;
builder.factorize(A);
builder.solve(...);
}
Examples:
demo_complex.cpp, demo_fgmres.cpp, demo_fgmres_sparsifier.cpp, demo_gmreshif.cpp, demo_hifir.cpp, demo_mixedprecision.cpp, demo_multiply.cpp, demo_params.cpp, demo_simple.cpp, demo_sparsifier.cpp, demo_trans.cpp, and demo_verbose.cpp.

factorize the HIF preconditioner with generic interface

Template Parameters
IsCrsif true, then the input compressed structure will be assumed to be CRS format
IndexType_integer data type, e.g., int
ValueType_value data type, e.g., double
Parameters
[in]nsize of system
[in]indptrindex starting position array, must be length of n+1
[in]indicesindex value array, must be length of indptr[n]
[in]valsvalue array, same length as that of indices
[in]paramscontrol parameters, using the default values.
[in]m0leading block size, if it's zero (default), then the routine will assume an asymmetric leading block.

This interface differs from the above one is that it takes plain-old-data types as input thus flexible. Notice that the integer and floating data types don't need to align with index_type and value_type, which aims for mixed-precision computation.

get the stats

Parameters
[in]entrystatistic entry

Currently, the following stats are available:

  • 0: total deferrals
  • 1: dynamic deferrals
  • 2: dynamic deferrals due to bad diagonals
  • 3: dynamic deferrals due to bad conditioning
  • 4: total droppings
  • 5: droppings due to space-controling Notice that entry 0 subtract entry 1 is the static deferrals, which are due to zero diagonals that occurs when applying symmetric preprocessing.
