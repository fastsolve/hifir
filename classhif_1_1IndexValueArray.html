Detailed Description

template<class ValueType, class IndexType>
class hif::IndexValueArray< ValueType, IndexType >

Template Parameters
ValueTypevalue type, e.g. double, float, etc
IndexTypeindex type, e.g. int
Note
We use std::vector for internal data management

This data structure stores two essential arrays: 1) a dense array of application "values" and 2) a sparse array of the activated indices in the value array. By dense, we mean an array that supports random access; by sparse, we mean an array that only contains what we need. Notice that this data structure is mainly used in sparse matrix operations for efficiency, where we just need to manipulate the indices without worry about touching the values. Be aware the it's may be necessary that the sparse array have the same length (in terms of memory) as that of the dense array due to the fact that we cannot estimate the sparse size precisely.

For the efficiency purpose, we explicitly create an size counter, _counts, to avoid calling the std::vector::push_back, which is quite expansive.
