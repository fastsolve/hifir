LhfStatus lhfzApply ( const LhfzHifHdl  hif,
const LhfOperationType  op,
const double _Complex *  b,
const int  nirs,
const double *  betas,
const int  rank,
double _Complex *  x,
int *  ir_status 
)

Apply a preconditioner with a certian operation mode.

Parameters
[in]hifA HIF instance
[in]opOperation tag
[in]bThe RHS vector
[in]nirsNumber of iterative refinements
[in]betasRelative residual norm bounds for IR with op=solve
[in]rankNumerical rank
[out]xComputed solution vector
[out]ir_status(optional) If IR and betas is provided, then this records the number of actual IR ( ir_status[0] ) and the IR status ( ir_status[1] ).

This is the core function of HIF. A HIF instance can be applied in four different modes, namely triangular solve and its Hermitian/tranpose variant, as well as matrix-vector multiplication and its Hermitian/tranpose variant. For the triangular solve modes, nirs can be greater than one, which means iterative refinement is enabled. With IR enabled, if betas is provided, i.e., not NULL, then we will iterative until (1) res<=betas[0], (2) res>betas[1], or (3) it=nirs, and the actual iteratoin steps and return status are stored in ir_status if provided (i.e., not NULL ). On the other side, if betas=NULL, then the a fixed amount of IR ( nirs ) are performed and ir_status is not accessed. Regarding rank, in general, it should use LHF_DEFAULT_RANK.

LhfzHifHdl lhfzCreate ( const LhfzMatrixHdl  A,
const LhfzMatrixHdl  S,
const double  params[] 
)

Create a double-precision complex HIF instance.

Parameters
[in]AInput coefficient matrix (used in iterative refinement)
[in]SSparsifier input
[in]paramsControl parameters, see lhfSetDefaultParams

This function create an instance of double-precision HIF (see hif::HIF) preconditioner. Note that both A and S can be NULL, which then will create an empty instance. If both operators are provided, then the preconditioner will be factorized based on S. If only A is provided, then HIF is computed on A. If only S is provided, then A is set to be S besides computing the factorization on it. In addition, if params is NULL, then the default parameters will be used.

◆ lhfzGetStats()

LhfStatus lhfzGetStats ( const LhfzHifHdl  hif,
size_t  stats[] 
)

Get the statistics of a computed HIF instance.

Parameters
[in]hifA HIF instance
[out]statsA length-9 array stores certain useful information

Regarding the output stats:

  • stats[0]: Number of nonzeros of a preconditioner
  • stats[1]: Total deferals
  • stats[2]: Dynamic deferals (status[1]-status[2] are static deferals)
  • stats[3]: Total droppings
  • stats[4]: Droppings due to scalability-oriented strategy
  • stats[5]: Number of levels
  • stats[6]: Numerical rank of the whole preconditioner
  • stats[7]: The numerical rank of the final Schur complement
  • stats[8]: The size of the final Schur complement (status[7]<=status[8])
