Detailed Description

filter attached to HIF for filtering null space components

Flexible design of null space components filter. The default mode is constant filter, i.e., there is a constant mode in the system. Two ways to customize the filter, 1) override the NspFilter::user_filter function or 2) pass in a callback function.

For both 1) and 2), the interface is void(void*,std::size_t,char) where the first parameter is in/out array whose length is specified by the second parameter. The last parameter indicates the data types, which follow the BLAS conventions, i.e., 'd' for double, 's' for single, etc.

As to the user call back function, it is wrapped inside std::function, which allows easy binding external user data. For more, see std::bind.

In most cases, a (partial) constant mode is enough. If user-customized behaviors are needed, then it will be easy to work with user callback in C++ and override NspFilter::user_filter in other programming environments, e.g., Python.
